Adding columns to your board defines the stages of your workflow, such as "Todo", "In Progress", or "Done", allowing you to categorize and track tasks effectively. You can add columns during board creation or when editing an existing board via the board setup modal. There is no direct "+ Add Column" button on the active board view itself—columns must be managed through board creation or editing.

## Column Display on the Board
When viewing an active board, each column appears as a vertical panel (fixed width) containing:
- A colored dot indicator (automatically assigned based on column position: first column uses one color, second another, and so on from a predefined palette).
- The **column name** (uppercase, bold, small text; truncated if longer than 10 characters, with task count in parentheses, e.g., "TODO (3)").
- Tasks listed within the column (tasks can be dragged between columns).
- Icons next to the name:
  - **Pencil icon** (**IoPencil**): Click to enter edit mode for the column name.
  - **Delete icon** (**AiOutlineDelete**, red-tinted): Click to open a confirmation popup for deleting the column (removes the column and all its tasks).
- In edit mode: An input field replaces the name (pre-filled with current name, bordered red if invalid), with **X icon** (close/cancel) and **checkmark icon** (save).

> [!NOTE]
> Column order is fixed based on creation sequence and cannot be rearranged directly. Colors are assigned automatically by position.

## Adding Columns During Board Creation
Follow these steps to include columns when setting up a new board:

1. Navigate to create a new board (see 4.1. Creating a Board).
2. In the **Add New Board** modal, fill the **Name** field for the board.
3. Scroll to the **Columns** section (labeled in bold small text).
4. Existing column inputs (if any) appear as text fields with delete options—each starts empty.
5. Click the **+ Add New Column** button (full-width, styled button below the column list).
6. A new empty input field appears for the column **name**.
7. Enter a name (e.g., "Todo"), then repeat steps 5-6 for additional columns.
8. Click **Create Board** at the bottom.

Expected result: The board is created with your specified columns, each starting with 0 tasks. A toast notification appears briefly if the board name duplicates an existing one ("Board already exist.")—change the board name and try again.

| Field | Description | Required | Accepted Values | Behavior on Change |
|-------|-------------|----------|-----------------|---------------------|
| **Name** (per column input) | Label for the workflow stage, shown on the board with task count. | Yes | Text string (e.g., "Todo", "Progress"); no specified length limit, but keep short for display. | Updates the column header instantly on save; empty fields trigger validation error ("Required"). |

> [!WARNING]
> At least one column is required, or submission shows "Add a column." below the button. Column names do not have uniqueness checks within a board.

## Adding Columns to an Existing Board
To add columns after creation:

1. Open the board you want to modify.
2. Click the edit board control in the header (see 3.2. Header Controls or 4.2. Editing and Deleting Boards).
3. The **Edit Board** modal opens, pre-filled with current board name and columns.
4. In the **Columns** section, click **+ Add New Column**.
5. Fill the new **name** field.
6. Add more columns as needed, or delete extras using the delete control on any input.
7. Click **Update Board** at the bottom.

Expected result: The board refreshes with the new column(s) appended to the end, retaining existing tasks in prior columns. New columns show "(0)" task count.

## Related Features
- **Tasks**: New columns start empty; add tasks via **+ Add Task** in a column (see 6.1. Adding Tasks). Tasks can be dragged to new columns.
- **Editing Column Names**: Use the pencil icon directly on the board (see 5.2. Editing Column Names).
- **Deleting Columns**: Use the delete icon on the board (removes column and tasks; confirmation required).
- **Board Management**: Columns are tied to boards—deleting a board removes all its columns (see 4.2. Editing and Deleting Boards).

> [!NOTE]
> Changes to columns (add, edit, delete) persist across sessions and sync automatically. If a column name exceeds display width, it truncates with ellipsis.
