Editing column names allows you to customize the labels for columns in your active board, such as changing "Todo" to "Backlog" for better organization. This inline editing feature provides quick updates without leaving the board view, with built-in validation to prevent empty names that could disrupt your workflow.

## Accessing Column Editing
Column editing is available directly on the active board. 
- Navigate to your board via the **Sidebar Board List** 3.1. Sidebar Board List or **Header Controls** 3.2. Header Controls.
- Each column header displays: a colored dot marker, the column *name* (truncated if longer than 10 characters), the task count in parentheses (e.g., "Todo (3)"), a **pencil icon** for editing, and a **delete icon** for removal.

## Step-by-Step: Editing a Column Name
1. Locate the column header you want to edit on the active board.
2. Click the **pencil icon** next to the column name.
3. The name changes to an editable **text input field** pre-filled with the current name. Two buttons appear: a **close icon** (X) on the left and a **checkmark icon** (✓) on the right.
4. Type your new name in the **text input field**. The field accepts any text up to a practical length (long names truncate in the header view).
5. Click the **checkmark icon** to save changes. The board updates immediately with the new name across all views.
6. To cancel, click the **close icon**; the input reverts to the original name without changes.

## Edit Interface Elements
| Element | Description | Action |
|---------|-------------|--------|
| **Pencil icon** | Triggers inline editing mode for the specific column. | Click to enter edit mode. |
| **Text input field** | Replaces the column name during editing. Border turns *red* if validation fails. | Type the new name; changes update live as you type. |
| **Close icon** (X, red) | Cancels editing and discards changes. | Click to exit without saving. |
| **Checkmark icon** (✓, green) | Applies the new name if valid. | Click to save and exit edit mode. |
| **Delete icon** (trash) | Opens column deletion (covered in other sections; not part of editing). | Avoid during editing to prevent conflicts. |

## Column Name Field Details
| Field Name | What it Represents | Required? | Accepted Values | Behavior on Change |
|------------|--------------------|-----------|-----------------|---------------------|
| **Text input field** (unnamed in UI, shows current name) | The display label for the column, used in headers, task assignment, and filtering. | Yes | Any text string (letters, numbers, spaces, symbols); no strict length limit but truncates over 10 chars in header. | Updates preview live; empty values trigger error on save attempt. |

## Validation and Error Handling
- **Empty name validation**: Saving an empty field (deleted all text) highlights the **text input field** with a *red border*. No save occurs; fill in a name and try again.
- No other validations (e.g., duplicates) apply during inline edits—only non-empty is enforced.
- Expected result: Successful save renames the column instantly; task counts and positions remain unchanged. The board refreshes to show the new name.
- Errors are visual only (red border); no popups or toasts for this feature.

> [!NOTE]  
> Edited names sync across the board and persist until you edit again or recreate the board. Use descriptive names like "In Review" for clarity in team workflows.

> [!WARNING]  
> Avoid very long names (>20 chars) as they may truncate in mobile views or lists. Edit during low-activity to prevent drag-and-drop interruptions.
