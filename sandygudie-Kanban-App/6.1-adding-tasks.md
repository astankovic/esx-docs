Adding tasks is essential for building out your board's workflow, allowing you to create new items in specific columns or edit existing ones. This feature opens a dialog where you input task details like title, description, subtasks, deadlines, and column assignment. It prevents duplicate task titles within the same board and handles moving tasks between columns during edits.

## Ways to Access the Add Task Dialog
You can open the dialog in these common ways, each pre-filling certain details:
- **From a column popup**: Right-click or use the **+** button on a column header to add a task directly to that column (column is pre-selected and locked).
- **From the header button**: Click the **+ Add Task** button in the board header to add a new task and choose any column.
- **From an existing task**: Click the edit icon (pencil) on a task card to open the dialog pre-filled with that task's details for editing.

## Dialog Layout and Controls
The dialog appears as a modal overlay with a scrollable content area. At the top is a bold heading:
- *"Add New Task"* for new tasks without a pre-selected column.
- *"Add [Column Name] Task"* for tasks tied to a specific column.
- *"Edit Task"* for modifying an existing task.

Below the heading:
- Form fields for task details (detailed in the next section).
- A **+ Add New Subtask** button (styled as a full-width rounded pill) to dynamically add subtask rows.
- **Create Task** or **Update Task** submit button at the bottom (full-width, primary color, hover effect).
- An implicit **Cancel** or **X** close button (via **handleClose**) in the dialog header or backdrop click.

## Task Fields
All fields are user-editable unless noted. Required fields show inline validation errors (red text below the field) if invalid, such as missing data.

| Field Name | Description | Required? | Accepted Values/Format | Default | Behavior on Change |
|------------|-------------|-----------|-------------------------|---------|--------------------|
| **Title** | Short name for the task, e.g., "Pending design task". | Yes | Any text (no length limit specified). | Empty | Updates the task name; checked for duplicates board-wide on submit. |
| **Description** | Detailed notes, e.g., "The hero page design is not completed". | No | Multi-line text. | Empty | Expands the textarea as you type. |
| **Deadline** | Due date for the task. | No | Date picker (YYYY-MM-DD format). | None | Sets a calendar date; combines with **Time** for full deadline. |
| **Time** | Due time on the deadline date. | No | Time picker (HH:MM format, 24-hour). | None | Sets clock time; pairs with **Deadline** field. |
| **Column** | The column (status) where the task belongs. | Yes (pre-filled) | Dropdown list of all column names on the board. Locked text input if opened from a column popup. | Pre-selected based on access method (e.g., first column or source column). | Changes task placement; during edits, moving columns relocates the task. |
| **Subtasks** | List of child items under the task (e.g., "Add company logo"). | Yes (at least one with a title) | Dynamic rows, each with a **title** text field and completion checkbox. | Empty list | Error shows if none added: "Add a substask." Each row has a delete button (via subtask input controls). |

> [!NOTE]  
> Subtasks must have titles to submit. Empty subtasks are invalid, but you can add/remove them freely before submitting.

## Step-by-Step: Adding a New Task
1. Open the dialog via column popup, header button, or board navigation.
2. Fill in **Title** (required).
3. Optionally add **Description**, **Deadline**, and **Time**.
4. Click **+ Add New Subtask** to create at least one subtask row; enter a title for each and check *completed* if done.
5. Select a **Column** from the dropdown (if not locked).
6. Click **Create Task**.
7. The task appears instantly in the chosen column as a new card. Dialog closes.

Expected result: Task card shows title, description preview, completed subtasks count, and deadline/time if set.

## Step-by-Step: Editing a Task
1. Click the edit icon on any task card.
2. Modify fields as needed (e.g., update title, add subtasks, change **Column**).
3. Click **Update Task**.
4. Changes save immediately:
   - If column unchanged: Task updates in place.
   - If column changed: Task moves to the new column.
5. Dialog closes; refreshed card reflects updates.

Expected result: Updated task card in the (possibly new) column, with revised details visible on hover or click.

## Error Messages and Troubleshooting
- **Inline field errors** (red text below field): "Required" for empty **Title** or subtask titles; "Add a substask." if no subtasks. Fix by filling required data.
- **Toast notification** (top of screen, red, auto-dismisses after 2 seconds): "Task already exist." Appears if the title matches any existing task on the board (case-sensitive?). Change the title and retry.
- Form won't submit until validation passes (e.g., no empty required fields).

> [!WARNING]  
> Duplicate titles are checked across the entire board, not per column. Use unique titles to avoid blocks.
