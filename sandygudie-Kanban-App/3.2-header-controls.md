The Header Controls bar, fixed at the top of the screen, offers essential shortcuts for managing your workspace, creating tasks, customizing the display, and switching themes. It appears on every dashboard view once a board is active, providing one-click access to key actions without navigating away from your work.

## Layout and Visible Elements
The header spans the full width of the screen and includes these main sections from left to right:

- **Logo**: A branded icon on desktop (hidden on mobile) or a compact logo image on mobile devices. Clicking it has no action but serves as a visual anchor.
- **Workspace selector**: Displays the current workspace *name* (truncated if longer than 10 characters) followed by a downward chevron icon.
- **Right-side controls** (spaced evenly):
  - **Fullscreen button**: Shows a screen-expand icon when not in fullscreen or a zoom-out icon when active.
  - **Theme toggle**: A switch control for light/dark modes.
  - **+ Add Task button**: A circular button with a plus icon (mobile) or "+ Add Task" text (desktop). Only visible when a board with at least one column is active.

> [!NOTE]  
> On mobile devices (screens under medium width), the desktop logo hides, the workspace name may truncate further, and the Add Task button shows only the plus icon for space efficiency.

## Workspace Menu
Clicking the **workspace name** button opens a dropdown popup positioned below it, showing your current workspace details and management options. The popup includes:

- A profile preview panel at the top with:
  | Element | Description |
  |---------|-------------|
  | Workspace image | A placeholder avatar (8x8 pixels). |
  | Workspace *name* | Bold, large text (e.g., "My Workspace"). |
  | Email | Smaller text below the name (e.g., "user@example.com"). |

- A list of actionable menu items below the preview:

| Menu Item | Action | Availability |
|-----------|--------|--------------|
| **Invite people to [Workspace Name]** | Opens a placeholder action (no effect yet). | Always available. |
| **Workspace settings** | Opens a placeholder action (no effect yet). | Always available. |
| **Switch workspace** | Navigates to the workspace selection screen. | Always available. |

### Opening the Workspace Menu
1. Locate the **workspace name** with the chevron at the top-left of the header.
2. Click it to toggle the popup open.
3. Select an item from the list to perform the action.
4. Click outside the popup or select an item to close it automatically.

This connects to broader workspace management, such as switching boards via 2.2. First Workspace Setup or future invites/settings.

## Fullscreen Toggle
The **fullscreen button** expands the entire app to fill your screen, hiding browser chrome for immersive viewing of boards and tasks.

### Toggling Fullscreen Mode
1. Click the **fullscreen button** (screen-expand icon).
2. The app enters *fullscreen mode*, and the icon changes to a zoom-out icon.
3. Click again to exit fullscreen and restore normal view.

- **Expected result**: In fullscreen, all distractions vanish; exit restores the standard window.
- **Browser support**: Works in modern browsers; if unavailable, the button has no effect.

> [!WARNING]  
> Fullscreen may not persist across tabs or browser restarts—use it for focused sessions only.

## Theme Toggle
The **theme toggle** switch lets you instantly switch between *light* and *dark* modes, with your preference saved automatically for future sessions.

| Setting | Description | Default |
|---------|-------------|---------|
| **Light** | Bright background, dark text for daytime use. | Stored preference or *dark*. |
| **Dark** | Dark background, light text for low-light environments. | *Dark* if no preference set. |

### Changing the Theme
1. Locate the **theme toggle** switch in the header's right side.
2. Slide or click to switch modes.
3. The entire interface updates immediately, and the change persists on reload.

This affects all UI elements app-wide, including boards and modals, for consistent viewing.

## Add Task Button
When viewing an active board with columns, the **+ Add Task** button provides instant access to task creation without dragging or sidebar navigation.

### Adding a Task from the Header
1. Ensure a board is loaded with at least one column (see 3.1. Sidebar Board List).
2. Click **+ Add Task**.
3. A full-screen overlay modal opens with a task creation form.
4. Fill in task details and submit (detailed in 6.1. Adding Tasks).
5. The modal closes, and the new task appears in the first column.

- **Visibility**: Hidden if no board is active or no columns exist—activate a board first via the sidebar.
- **Expected result**: Quick task entry directly from your workflow, integrating with 5. Managing Columns and 6. Creating and Managing Tasks.

## Troubleshooting Header Issues
- **Add Task button missing**: Load a board with columns via 3.1. Sidebar Board List.
- **Workspace menu not opening**: Ensure you're on the dashboard (not setup screens); refresh if stuck.
- **Theme not saving**: Clear browser storage and reload—preferences are local to your device.
- **Fullscreen fails**: Check browser permissions; some extensions block it.
