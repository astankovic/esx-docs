The First Workspace Setup screen is your entry point when launching the app for the first time or when no workspace is configured. It guides you through creating a new workspace by providing your organization's name and email, or lets you view and access any existing workspace before proceeding to the dashboard. This setup is essential as it personalizes your Kanban experience and unlocks access to boards, tasks, and other features.

## Screen Layout and Elements
When you first access the app, you see a full-screen interface with these key elements:

- **Header bar** (top of the screen): Contains the app logo (desktop or mobile version), a title that reads **No Workspace** (if none exists) or **Workspace** (if one does), and a **theme toggle** switch on the right for switching between light and dark modes.
- **Available Workspaces section** (if a workspace exists): A centered panel displaying "Available Workspace(s)" with a clickable card showing a placeholder image, your workspace **name**, and **email**. Below it is an **Add Workspace** button.
- **Create Workspace form** (shown on first use or when adding a new one): A centered form panel with a welcome message "Welcome to Kanban! Get started by creating a workspace!" On desktop, promotional images slide automatically to the left. If adding to an existing workspace, a **back arrow button** labeled **Available workspace** appears at the top-left.

> [!NOTE]  
> The screen adapts to mobile devices, hiding the sliding images and using a compact logo.

## Viewing Available Workspaces
If you already have a workspace:

1. The screen shows your current workspace details in a clickable card.
2. Click the workspace card to proceed directly to the **Dashboard** 3. Dashboard and Navigation.
3. Click the **Add Workspace** button (blue, prominent at the bottom) to switch to the creation form.

## Creating a New Workspace
Use this when starting fresh or adding another workspace. The form validates inputs before submission.

### Form Fields

| Field Name          | Description                                                                 | Required? | Accepted Format/Values                  | What Happens on Change                  |
|---------------------|-----------------------------------------------------------------------------|-----------|-----------------------------------------|-----------------------------------------|
| **Organization's name** | Your workspace or company's name (e.g., "Acme Corp").                      | Yes      | Text, 5-25 characters only             | Updates the field; invalid lengths show error "At least 5 characters and not more than 25". |
| **Email**           | Contact email for the workspace (e.g., "team@acmecorp.com").                | Yes      | Valid email format                     | Updates the field; invalid format shows error "Required". |

- All fields start empty.
- Errors appear below each field if validation fails (e.g., missing, too short/long name, invalid email).
- **Create Workspace** button (full-width blue button at bottom): Submits the form. Disabled until valid.

### Step-by-Step: Create a Workspace
1. Ensure you're in create mode (automatically on first use, or click **Add Workspace** if viewing existing).
2. Fill in **Organization's name** (5-25 characters).
3. Enter a valid **Email**.
4. Click **Create Workspace**.
5. On success: The app saves your workspace and navigates to the **Dashboard** 3. Dashboard and Navigation, where you can create boards.
6. If errors: Fix highlighted fields and try again.

> [!WARNING]  
> Names shorter than 5 or longer than 25 characters prevent submission. Use a real email format to avoid validation blocks.

## Theme Toggle
- Located in the header (right side).
- Switches between *light* and *dark* modes.
- Persists your choice across sessions.
- Default: Matches your last used or *dark*.

## Expected Results and Navigation
- **Success**: Redirects to dashboard with your new workspace active. Future logins skip this screen.
- **Back navigation**: From create form (if existing workspace), click **Available workspace** arrow to return to the list view.
- This setup links directly to managing boards 4. Managing Boards once complete.

## Error Messages
Common issues and user-visible feedback:

| Error Message                          | Meaning                                      | How to Fix                          |
|----------------------------------------|----------------------------------------------|-------------------------------------|
| "Required" (under any field)           | Field left empty.                            | Enter a value.                      |
| "At least 5 characters and not more than 25" | Name too short or long.                      | Adjust to 5-25 characters.          |

## Log Messages
During operation, you might observe these messages in your browser console (press F12 to view). They indicate internal state for troubleshooting.

| Message                          | Severity | Meaning                                                                 |
|----------------------------------|----------|-------------------------------------------------------------------------|
| "true" (create workspace state)  | Info     | The app is actively showing the workspace creation form. No action needed. |
| "false" (create workspace state) | Info     | The app is showing the available workspaces view. No action needed.     |
