This section covers **Project Setup**, the process of using the Hono command-line interface (CLI) to generate a new project structure and automatically install dependencies. It is designed for users starting their first Hono web application from scratch, providing a ready-to-use template that supports multiple runtimes. This is the foundational step in [Getting Started](getting-started), directly leading into developing a [Basic Application](basic-application) and [Running the App](running-the-app). For broader context, see [Overview](overview); for runtime-specific details, refer to [Runtime Adapters and Deployment](runtime-adapters-and-deployment) including [Cloudflare Workers and Pages](cloudflare-workers-and-pages) and [Server Runtimes (Deno, Bun, Node)](server-runtimes-deno-bun-node).

## Overview
The **Project Setup** feature uses a single CLI command to create a complete starter project. It generates essential files, configures package management, installs Hono and core dependencies, and sets up support for your selected runtime environment. This ultrafast process eliminates manual boilerplate, getting you coding immediately. The result is a new directory with a functional Hono app skeleton, ready for customization.

## Prerequisites
Before starting, ensure you have:
- A package manager like *npm* (included with Node.js) installed and accessible in your terminal.
- Write permissions in the target directory.

> [!NOTE]  
> Node.js version 18 or higher is recommended for full compatibility across runtimes.

## Step-by-Step: Scaffolding a New Project


1. Open your **terminal** or **command prompt**.
2. Navigate to the directory where you want the new project folder created (e.g., your workspace root).
3. Execute the project creation command: **npm create hono@latest**.
4. Respond to the interactive prompts:
   | Field | Required | Accepted Values | Description |
   |-------|----------|-----------------|-------------|
   | **Project name** | Yes | Alphanumeric string, hyphens allowed (*my-hono-app*) | Name of the new project directory; defaults to *hono-app* if blank. Creates a folder with this name. |
   | **Package manager** | Yes | *npm*, *yarn*, *pnpm*, *bun* | Tool for managing dependencies; selects matching lockfile and scripts. |
   | **Runtime** | Yes | *Node.js*, *Deno*, *Bun*, *Cloudflare Workers*, *Fastly Compute*, others | Target environment; configures adapters and entry points accordingly. |
   | **TypeScript** | No | *Yes* / *No* | Adds TypeScript configuration and *.ts* files if selected. |
5. Wait for the process to complete: the system creates the project folder, generates starter files (like entry points and configuration), and installs dependencies.
6. Navigate into the new project: **cd *your-project-name***.

The output confirms success with messages like "*Project created successfully*" and lists next steps.

> [!WARNING]  
> If the project name matches an existing directory, it may overwrite files—choose a unique name to avoid data loss.

## Installed Components
After setup, your project includes:
- Core Hono library for routing and middleware.
- Runtime-specific adapters (e.g., for Cloudflare Workers).
- Package configuration with install scripts.
- Basic app entry file ready for editing.

## Troubleshooting
Common issues and resolutions:

| Message | Severity | Meaning |
|---------|----------|---------|
| *npm: command not found* | Error | Package manager missing; install Node.js from the official site and retry. |
| *EACCES: permission denied* | Error | Insufficient directory permissions; run terminal as administrator or change to a writable folder. |
| *Network error fetching template* | Warning | Temporary connectivity issue; check internet and retry the command. |
| *Project name already exists* | Error | Directory conflict; choose a different name or remove the existing folder. |

If prompts do not appear, ensure your terminal supports interactivity (avoid piping input).

## Summary
- Use **npm create hono@latest** for instant project scaffolding with dependency installation.
- Customize via interactive prompts for **project name**, **package manager**, **runtime**, and **TypeScript**.
- Results in a runtime-ready folder structure for immediate development.
- Follow up with [Basic Application](basic-application) for your first routes, [Running the App](running-the-app) to start the server, and [Routing](routing) for advanced paths.
- For deployment, see [Runtime Adapters and Deployment](runtime-adapters-and-deployment). Refer to [Configuration Reference](configuration-reference) for custom tweaks.
