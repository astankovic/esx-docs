This section covers starting the development server to run your Hono application locally and testing your initial endpoints in a browser or via command-line tools. It's for users following the [Getting Started](getting-started) guide who have completed 2.1. Project Setup and 2.2. Basic Application. Once the server is running, you can verify your app responds correctly before exploring advanced features like 3.1. Basic and Parametric Routes or 4. Middleware.

## Overview
The development server creates a local web environment where your app's endpoints become accessible via a browser or HTTP client. It displays startup messages in the terminal, including the local address (typically *http://localhost:8787*), and supports hot reloading for quick iterations during development. Key capabilities include one-command startup, automatic port binding, and graceful shutdown.

## Starting the Development Server
Follow these steps to launch the server from your project directory:

1. Open a terminal or command prompt.
2. Navigate to your project folder if not already there.
3. Run the **npm run dev** command (or the equivalent **dev** script listed in your project tools).
4. Observe the terminal output confirming the server is ready, such as:
   - *Server listening on http://localhost:8787*
   - *Development mode enabled with hot reload*
5. The server runs continuously until you stop it.

> [!NOTE]  
> Keep the terminal open while the server runs. Changes to your app may trigger automatic reloads in development mode.

To stop the server:
1. In the terminal, press **Ctrl+C** (or **Cmd+C** on macOS).
2. Confirm the shutdown message, such as *Server stopped*.

## Testing Endpoints
With the server running, interact with your app's endpoints to confirm they work:

1. Open a web browser.
2. Enter the local address shown in the terminal (e.g., *http://localhost:8787/*).
3. Verify the response from your root endpoint, such as a simple welcome message.
4. Test additional endpoints by appending paths (e.g., *http://localhost:8787/api/users* if defined).
5. Use command-line tools for non-browser tests:
   - Run **curl http://localhost:8787/** in another terminal to fetch the root response.
   - Check status codes, headers, and body content in the output.

> [!TIP]  
> Browser developer tools (Network tab) show request details like response time and status. For POST requests or JSON, use tools like Postman.

If an endpoint returns unexpected results:
- Refresh the browser or re-run the curl command.
- Check the terminal for any runtime notices.

## Summary
- Launch the development server with **npm run dev** to access your app at a local URL like *http://localhost:8787*.
- Test endpoints directly in a browser or with **curl** to validate responses.
- Stop the server anytime with **Ctrl+C**; hot reload speeds up development.
- After testing basics, proceed to 3. Routing for more endpoint types or 8. Runtime Adapters and Deployment for production.

For project initialization, review 2.1. Project Setup. For app structure, see 2.2. Basic Application.
